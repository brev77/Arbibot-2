# Полный справочник операторского интерфейса Arbibot 2

Документ для новичков: что есть на каждой странице, какие **поля ввода**, **кнопки** и **ссылки** встречаются, и зачем они нужны. Внизу — **глоссарий с расшифровкой сокращений**.

**Как устроен доступ к данным:** браузер почти никогда не ходит напрямую в микросервисы. Запросы идут в **BFF** (см. глоссарий) под путём `/api/operator/…` на том же сайте, что и интерфейс. Секреты и ключи API остаются на сервере Next.js.

**Роли (RBAC — см. глоссарий):** `viewer` < `operator` < `admin`. Минимальная роль для раздела задаётся в коде. В разработке роль можно задать переменной окружения `ARBIBOT_DEV_ROLE` или cookie `arbibot_role`.

### Панель фильтров (`OperatorFiltersBar`)

| Элемент | Тип | Назначение | Влияние |
|--------|-----|------------|---------|
| **Search** (поиск) | поле ввода | Подстрока для фильтрации **списка возможностей** на странице `/opportunities` (по id, correlation, risk decision) | Только отображение на клиенте |

### Баннер деградации intake (`DegradedStatusBanner`)

| Элемент | Тип | Назначение | Влияние |
|--------|-----|------------|---------|
| Текст баннера | информация | Опрос каждые 30 секунд: `GET /api/operator/health/degradation` (рынок intake). Показывает tier, fallback, включён ли throttling, оценку частоты throttle | Нет (только индикация) |
| Состояние ошибки | текст | Если запрос не удался — красное сообщение «Intake status unavailable» | Нет |

### Баннер Safe mode (`SafeModeBanner`)

| Элемент | Тип | Назначение | Влияние |
|--------|-----|------------|---------|
| Красная полоса | информация | Опрос HERMES: включён ли safe mode. Показывается только если включён | Нет (только индикация) |

Если gateway не настроен или запрос падает — баннер не показывается.

### Компонент опасных действий (`DestructiveOperatorAction`)

Встречается на **Incidents**, **Settings**, **HERMES** и др. Это **не одна кнопка**, а поток:

1. Кнопка с подписью действия (например «Apply update»).
2. Для уровней **medium** и **high** открывается окно с **превью последствий** (что затронуто, риски, смягчение).
3. Для **high** и заданной фразы нужно **ввести точную фразу** (например `APPROVE`, `ARM`, `EXECUTE`, `CLOSE`) и подтвердить второй шаг.

**Зачем:** снизить риск случайного клика по опасным операциям.

## Страница `/` (домашняя)

В **production** без сессии оператора показывается сообщение «Operator access unavailable» с подсказкой про `ARBIBOT_DEV_ROLE` / cookie.

Если в адресной строке есть `?forbidden=1` (после отказа в доступе) — страница **Access denied** с пояснением требуемой роли и ссылками **Home** и **Try the dashboard**.

Если сессия есть (или не production) — выполняется **перенаправление** на `/dashboard` (отдельной формы нет).

## Глоссарий сокращений и терминов

| Сокращение / термин | Расшифровка / значение |
|---------------------|-------------------------|
| **API** | Application Programming Interface — программный интерфейс между системами (набор URL и правил обмена данными). |
| **BFF** | Backend-for-Frontend — слой на сервере приложения (здесь Next.js), который прячет секреты и ходит в микросервисы от имени UI. |
| **RBAC** | Role-Based Access Control — доступ по **ролям** (набор прав), а не отдельно на каждого пользователя. |
| **HTTP** | Hypertext Transfer Protocol — протокол, по которому браузер и сервер обмениваются запросами и кодами (200 OK, 401, 503 и т.д.). |
| **JSON** | JavaScript Object Notation — текстовый формат данных `{ "key": "value" }`. |
| **REST** | Representational State Transfer — стиль API поверх HTTP (ресурсы, методы GET/POST/PATCH…). |
| **URL** | Uniform Resource Locator — адрес страницы или API в интернете. |
| **UI** | User Interface — интерфейс пользователя (экраны, кнопки). |
| **RSC** | React Server Components — компоненты React, выполняемые на сервере (например карточка возможности `/opportunities/[id]`). |
| **TTL** | Time To Live — время жизни кэша: как долго сервис может использовать старое значение конфига до следующего запроса. |
| **USD** | United States Dollar — доллар США. |
| **UTC** | Coordinated Universal Time — всемирное координированное время (метки «Recorded (UTC)» в таблицах). |
| **bps** | Basis points — базисные пункты; 1 bps = 0.01% (используется для дрейфа цен). |
| **ms** | Milliseconds — миллисекунды (интервалы в конфигах). |
| **CFG-3** | Третья итерация функционала **configuration** в проекте: черновики, промоут между scope, активация. |
| **scope** | Область действия конфигурации: **global** (везде), **environment** (среда, например staging), **tenant** (конкретный арендатор). |
| **effective** | Итоговое значение ключа после цепочки fallback: обычно tenant → environment → global. |
| **outbox** | Паттерн записи событий в БД для последующей надёжной доставки в шину сообщений. |
| **single-writer** | Принцип: одна служба «владеет» записью сущности, остальные только читают или шлют команды ей. |
| **Prometheus** | Система метрик; имена вроде `arb_paper_drift_*` — для дашбордов и алертов. |
| **Phase 3 / 4 / 5** | Этапы дорожной карты продукта: бумажная торговля, деградация intake / политики, HERMES и т.д. |
| **NestJS** | Фреймворк для серверных приложений на TypeScript (микросервисы Arbibot). |
| **Next.js** | Фреймворк для веб-приложений React (операторский UI). |
| **PostgreSQL** | Реляционная база данных (OLTP — Online Transaction Processing: операционные транзакции). |
| **Zod** | Библиотека проверки структуры JSON в TypeScript (валидация политик в формах Settings). |
| **correlationId** | Идентификатор для связи нескольких событий/запросов одной цепочки. |
| **notional** | Условная сумма позиции в деньгах (номинал), для оценки экспозиции. |
